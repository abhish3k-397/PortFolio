# Design Context

## Users
- **Primary**: Mixed audience - technical recruiters hiring for software roles AND developer community peers
- **Context**: Recruiters view on desktop during candidate review; developers explore on various devices
- **Job to be done**: Showcase technical skills, projects, and personality to land interviews

## Brand Personality
- **Three words**: Technical, Distinctive, Memorable
- **Voice**: Confident, cutting-edge, authentic (not generic)
- **Emotional goal**: Make visitors think "this developer is different from the rest"

## Aesthetic Direction

| Theme | Purpose | Visual Style |
|-------|---------|--------------|
| **Cyberpunk** | For the community/developers | Aggressive neon (red #ff003c, yellow #fcee0a), glitch effects, terminal aesthetics, "out of the box" |
| **Ink & Paper** | For recruiters/professionals | Clean, Japanese minimalism, warm cream backgrounds, vermillion accents, calming, serif typography |

**Anti-references**: Generic SaaS landing pages, bento grid glassmorphism (AI slop), purple/gradient-heavy designs

## Design Principles

1. **Dual-track appeal**: Cyberpunk theme serves the developer community with bold, distinctive aesthetics; Ink & Paper serves recruiters with clean professionalism
2. **Motion with purpose**: Animations should feel intentional, not decorative — every effect should serve the narrative
3. **Easter eggs reward curiosity**: Hidden features (terminal, hacking game, achievements) create delight for explorers
4. **Performance is a feature**: Complex animations must not sacrifice load times
5. **Accessibility without compromise**: Bold themes should still work for users with visual differences
